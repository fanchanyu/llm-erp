# LLM-ERP V2 Roadmap — 組織驅動的智能工廠

> 從「LLM 聊天查詢 ERP」進化到「LLM 驅動的智能工廠營運」

---

## 🎯 V2 核心命題

V1 證明了 LLM 可以自然語言查詢與操作 ERP。V2 要回答的是：

> **當真實使用者（經理、組長、採購、倉管）透過 SSLVPN/內網登入系統時，系統如何理解「誰在做什麼、能看到什麼、能批准什麼」？**

---

## 🔴 Phase 0 — 組織人事基礎（最優先）

**目標：讓系統可以真正開放給工廠人員使用**

### ① 部門管理
- 多層級組織樹（總公司 → 廠區 → 部門 → 課組）
- 部門主管、代理人設定
- 部門狀態（啟用/停用）

### ② 員工管理
- 員工基本資料（工號、姓名、職稱、到職日、email）
- 員工 ↔ 系統帳號 1:1 對應
- 在職/離職狀態管理

### ③ 權限系統 (RBAC + Department + Data)

| 權限層級 | 說明 | 範例 |
|---------|------|------|
| **角色權限** | 職位決定的橫向功能 | 操作員看庫存、經理可下採購單 |
| **部門權限** | 水平資料隔離 | 倉管課只能看自己倉庫的庫存 |
| **資料權限** | 欄位/單據級別控制 | 只看得到金額前 3 碼、報價單需簽核 |
| **特殊權限** | 覆寫例外 | 廠長可跨部門查詢 |

### ④ 簽核流程
- 採購單簽核、報價簽核、請假簽核、入庫簽核
- 多層簽核（課長→經理→廠長）
- 代理簽核機制
- 簽核歷程記錄

### ⑤ 登入與稽核
- JWT 認證 + Session 管理
- 登入紀錄（IP、時間、裝置）
- 操作 Audit Log（誰、何時、做了什麼、改了什麼）
- 登入失敗鎖定機制

---

## 🟡 Phase A — 核心製造流程深化

**目標：從「查得到生產進度」到「系統幫你排生產」**

### ⑥ MPS/APS 排程引擎
- 主生產排程 (MPS)
- 進階排程與規劃 (APS) — 產能限制、順序排程
- 可視化甘特圖
- 與現有 Dispatch 模組整合

### ⑦ 工單管理 (Work Orders)
- 從銷售訂單自動產生工單
- 工單派工、完工回報
- WIP (在製品) 追蹤

### ⑧ 現場控制面板 (Shop Floor Control)
- 機台狀態監控
- 人員派工與負荷分析
- 即時生產進度看板

---

## 🟢 Phase B — 供應鏈與倉儲管理

**目標：從「知道庫存有多少」到「系統幫你管倉庫」**

### ⑨ 倉儲管理 (WMS)
- 儲位管理、庫別分類
- 揀貨、包裝、出貨流程
- 庫存異動（調撥、盤點、報廢）

### ⑩ 供應商管理
- 供應商評鑑（品質、交期、價格評分）
- 歷史報價、價格趨勢分析
- 自動補貨邏輯（安全庫存觸發 PO）

---

## 🔵 Phase C — LLM 智慧升級

**目標：從「被動回答問題」到「主動管理工廠」**

### ⑪ 多 Agent 協作
- 自動鏈：Lead → Opportunity → Contract → SO → Production → Shipment
- 採購 Agent 自動偵測庫存水位補貨
- 異常 Agent 主動告警（庫存不足、交期延遲、品質異常）

### ⑫ RAG 知識庫
- SOP 查詢、工程規範比對
- 歷史簽核案例檢索
- 與 Document Manager 整合

### ⑬ 語音介面
- 語音輸入查詢
- 事件的語音通知

### ⑭ 預測分析
- 需求預測（基於歷史銷售）
- 庫存水位預警
- 產能瓶頸預測

## 🏗️ DB Schema 策略

| 方案 | 說明 | 適用 |
|------|------|------|
| **從 V1 升級** | 在 V1 schema 上 + migration | 保留歷史資料，論文一致性 |
| **全新開始** | 重新設計 schema，V1→V2 不兼容 | Phase 0 改動太大時 |
| **混合** | V1 tables 不動 + 新增組織層 tables | 折衷方案，推薦 |

> 建議採用 **混合方案**：V1 的業務 tables（Inventory、Purchase、SO...）不動，新增 `departments`、`employees`、`roles`、`permissions`、`approval_flows` 等組織層 tables，透過 FK 串接既有資料。

## 📐 研究方向定義

### 研究方向一：多用戶協作 LLM-ERP 系統

**核心問題**：LLM 驅動的 ERP 系統如何在多用戶、多角色的組織環境中，透過自然語言介面實現協作？

| 面向 | 研究重點 |
|------|---------|
| 用戶 | 部門隔離、角色權限、資料權限如何影響 LLM 的回答 |
| 協作 | 多個 LLM Agent（採購、倉管、業務）如何協作完成跨部門任務 |
| 實證 | 對比有/無權限控制的 LLM 回答正確率與安全性 |

**期刊目標**：JMS (Journal of Manufacturing Systems) 或 CAIE (Computers & Industrial Engineering)

### 研究方向二：LLM-ERP 的組織層級授權框架

**核心問題**：提出一個基於 LLM function calling 的授權框架，讓使用者可以用自然語言發起需要多層簽核的請求。

| 面向 | 研究重點 |
|------|---------|
| 框架 | 定義 LLM Tool-based Approval Flow 模式 |
| 簽核 | 主管收到 LLM 通知 → 用自然語言批准/駁回 |
| 可追蹤 | 所有簽核過程自動記錄到 Decision Log |

**期刊目標**：Engineering Applications of Artificial Intelligence (EAAI)

### 研究方向三：簽核流程的視覺化管理

**核心問題**：如何讓使用者透過戰情室 (War Room) 視覺化地追蹤跨部門簽核流程？

| 面向 | 研究重點 |
|------|---------|
| 視覺化 | SVG-based 簽核流程圖，顯示卡在哪一關 |
| 互動 | 點擊節點可查看詳情、催辦、轉呈 |
| 整合 | 與 LLM 對話框整合，使用者用自然語言查詢簽核進度 |

**期刊目標**：IJPR (International Journal of Production Research)

## 📊 版本對照

| | V1 | V2 |
|--|-----|-----|
| **定位** | LLM 聊天查詢 ERP | 組織驅動的智能工廠 |
| **用戶** | 單一開發者/測試者 | 多部門、多角色工廠人員 |
| **認證** | 無 | JWT + RBAC + 部門隔離 |
| **簽核** | 無 | 多層簽核流程 |

| **倉儲** | 庫存查詢 | 完整 WMS |
| **供應商** | 無 | 評鑑+價格分析 |
| **Agent** | 單一 Orchestrator | 多域 Agent 協作 |
| **研究** | 可行性驗證 | 組織授權框架 |
