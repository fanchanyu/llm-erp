## 💡 學到的事 / 發現

```
1. LLM 預設會回退到最通用的工具 (query_inventory) — 需要明確的禁止規則才能糾正
2. JMS (IF 12.1) 對單一作者+無user study的論文太難 → 選 EAAI (IF 7.5) 更務實
3. "zero hallucination errors" 這類 claim 在學術論文是自殺行為 → 要改成謙卑語氣
4. DeepSeek 的 API 延遲 (4-12s) 是主要瓶頸，不是系統本身的問題
5. 跨模組查詢 (50%) > 單一模組 (100%) — 多層 agent 架構是下一步
6. IEEE Xplore / Elsevier 的網站會封鎖 headless browser
```

## ⚠️ 遇到的問題

| 問題 | 解決方式 |
|------|---------|
| Vite dev server 重啟前抓不到新加的 public/ 靜態檔 | 重啟 Vite 即可 |
| "/api/health" 路徑 vs "/health" 路徑不一致 | 改前端 fetch URL |
| AR API 回傳欄位名是 "items" 不是 "ar" | 修正 JS 取值路徑 |
| NC API 回傳是物件不是陣列 | 改 d?.ncs?.filter() |
| IEEE/Elsevier 封鎖 browser requests | 改用 Wayback Machine + 自身知識 |
| DeepSeek Vision 不吃 image_url | 無法用 vision_analyze（改用 MEDIA path） |
