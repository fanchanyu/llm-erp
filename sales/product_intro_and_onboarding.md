## 二、痛點陳述：中小製造業的ERP困境

### 現狀分析

| 問題 | 描述 | 影響 |
|------|------|:----:|
| **ERP太難用** | 鼎新、SAP介面複雜，功能選單上百層，員工抗拒使用 | 導入失敗率高達60% |
| **導入時間太長** | 傳統ERP導入需3-12個月，顧問費動輒數十萬 | 決策猶豫不決 |
| **訓練成本太高** | 每位員工需數週訓練，流動率高時成本加倍 | 教育訓練投入難以回收 |
| **Excel管理到極限** | 年營收5,000萬以上，Excel已無法滿足庫存精準度 | 庫存差異、出貨錯誤頻傳 |
| **老闆看不到即時數據** | 傳統ERP報表需IT人員協助，老闆無法隨手查詢 | 決策延遲、錯失商機 |
| **系統不聰明** | 傳統ERP只會記錄，不會分析、不會預警、不會建議 | 靠老師傅經驗，人走經驗亡 |

### 數字會說話

| 統計 | 數據 | 來源 |
|:---:|:----|:----:|
| 台灣中小製造業使用ERP比例 | ~45% | 經濟部中小企業處 |
| 導入ERP的中小企業中，滿意度<60%比例 | ~55% | 資策會MIC |
| 平均ERP導入週期（中小企業） | 4-6個月 | Gartner |
| 每年因庫存管理不當損失 | 營收的3-5% | 工研院IEK |
| 使用AI提升管理效率預期 | 30-50% | McKinsey |

## 四、關鍵差異化優勢

### vs 傳統ERP

| 維度 | 傳統ERP | LLM-ERP 🏆 |
|------|:-------:|:-----------:|
| **操作方式** | 點選單、填表單、記路徑 | **用說的/用打的，自然語言** |
| **學習時間** | 數天到數週 | **<30分鐘** |
| **導入時間** | 3-12個月 | **1-3天** |
| **查詢庫存** | 進多層選單→設過濾條件→看表格 | **「庫存量還有多少？」→ 直接回答** |
| **開採購單** | 開新單→填供應商→填品項→審核 | **「幫我開一張採購單給大明五金」→ 自動產生** |
| **月結作業** | 財務人員操作半天到一天 | **「幫我跑月結」→ 數分鐘完成** |
| **錯誤預防** | 欄位檢核（有限） | **AI語意檢查 + 20條約束規則 + 雙重確認** |
| **移動辦公** | App功能有限 | **LINE/Telegram都能用** |
| **報表分析** | 預設報表，需IT客製 | **用問的，AI即時產生報表** |
| **升級維護** | 需顧問、需停機 | **自動升級，零停機** |

### 核心差異點

1. **🇹🇼 繁體中文原生** — 支援台灣用語（「出貨單」、「廠商」、「沖銷」），不是簡轉繁
2. **🤖 多模型LLM** — 可切換OpenAI、Anthropic、DeepSeek或Ollama本地模型，不鎖定單一供應商
3. **🔒 安全雙重防護** — 20條業務規則約束 + 寫入雙重確認，AI不會亂操作
4. **⚡ 30分鐘Demo、1天上線** — 不到一杯咖啡時間就能展示完整價值
5. **📱 多平台** — Web、LINE、Telegram，員工用最習慣的方式操作
6. **🔄 自動迭代** — SaaS模式，每週更新，功能持續增加

## 五、快速上手路徑：1天上線

### 時間軸

```
註冊 → 匯入資料 → 設定 → 員工啟用 → 正式使用
 │        │        │        │          │
 10min    2hr     30min    1hr       即時
```

### Step 1：註冊開通（10分鐘）

1. 前往 [app.llm-erp.com]（示意網址）
2. 填寫公司基本資料（公司名稱、統一編號、電話）
3. 設定管理員帳號密碼
4. **選擇方案**：Starter免費試用14天（免信用卡）
5. 完成註冊 → 取得專屬登入連結

### Step 2：匯入資料（2小時，可遠端協助）

| 項目 | 準備內容 | 格式 |
|------|---------|:----:|
| **產品/物料主檔** | 料號、品名、規格、單位、單價 | Excel CSV |
| **庫存初始數量** | 料號、倉別、數量、單價 | Excel CSV |
| **BOM表** | 成品料號、組件料號、用量 | Excel CSV |
| **供應商** | 供應商名稱、統編、聯絡人、交期 | Excel CSV |
| **客戶** | 客戶名稱、統編、聯絡人、付款條件 | Excel CSV |
| **會計科目** | 科目代碼、科目名稱、類別 | Excel CSV |

> 💡 **我們提供資料匯入精靈**：上傳Excel，系統自動對應欄位，10分鐘驗證完成。也可以由我們的團隊協助匯入。

### Step 3：系統設定（30分鐘）

1. 設定倉庫/儲位（1-3個倉庫）
2. 設定使用者權限（誰可以查詢/新增/修改/刪除）
3. 設定通知規則（訂單確認、庫存預警、逾期提醒）
4. 設定審核流程（採購單、出貨單審核層級）
5. （選用）串接LINE/Telegram Bot

### Step 4：員工啟用與引導（1小時）

1. 新增使用者（匯入或手動新增）
2. 系統自動發出邀請信（含登入連結）
3. **內建引導機器人**：第一次登入時，AI會主動引導操作
4. **5分鐘教學影片**：每個模組一支短片
5. **預設10個常用問句**：員工可直接點選試用

### Step 5：正式使用（立即）

- 從第一天就開始用自然語言管理
- 範例操作：「查一下A倉庫還有多少螺絲M6」、「幫我開一張進貨單」
- 內建「並行模式」：可與現有系統同時使用，逐步轉移

## 八、潛在客戶FAQ

### 一般問題

**Q：LLM-ERP跟一般ERP有什麼不同？**
A：最大的不同是操作方式。傳統ERP需要點選單、填表單、記憶功能路徑；LLM-ERP用自然語言對話，您用中文說出需求，AI就會幫您完成。就像有個24小時的ERP助手隨時待命。

**Q：支援哪些語言？**
A：完全支援繁體中文（台灣用語），也支援英文。未來將擴展日文、越南文。

**Q：需要有AI或技術背景才能用嗎？**
A：完全不需要。我們的目標就是讓**沒有任何技術背景的現場人員**也能輕鬆使用。只要會打字或說話就行。

### 技術問題

**Q：資料安全嗎？**
A：我們提供兩層保障：
- Cloud方案：資料儲存在台灣本地AWS/GCP機房，符合個資法
- On-Premise方案：資料完全在公司內部，LLM可選本地模型
- 所有操作都有完整稽核軌跡

**Q：如果AI理解錯了怎麼辦？**
A：(1) 所有「寫入」操作都需要您的二次確認（Y/N）
   (2) 有20條業務規則自動檢查（例如：出貨數量>庫存量時會阻擋）
   (3) 您可以隨時修正：「不是這樣，改成...」
   (4) 每一筆操作都可回溯、可取消

**Q：跟我們現有的系統能整合嗎？**
A：Enterprise方案提供完整的REST API，可以跟CRM、MES、PLM、WMS等系統串接。我們也有標準資料匯入工具。
